# Sprint_5 — UI-автотесты для сервиса «Доска»

## Описание проекта

Проект содержит UI-автотесты для учебного сервиса [«Доска»](https://qa-desk.education-services.ru/).  
Тесты написаны на Python с использованием **Selenium WebDriver** и **pytest**.

## Структура проекта

```
Sprint_5/
├── locators/
│   ├── __init__.py
│   └── locators.py           # Локаторы элементов страницы
├── pages/
│   ├── __init__.py
│   ├── base_page.py          # Базовый класс с общими методами ожидания
│   ├── main_page.py          # Главная страница (аватар, кнопки входа/выхода)
│   ├── auth_page.py          # Формы авторизации и регистрации
│   └── ad_page.py            # Форма создания объявления и профиль
├── tests/
│   ├── __init__.py
│   ├── test_registration.py  # Тесты регистрации пользователя
│   ├── test_login.py         # Тесты авторизации
│   ├── test_logout.py        # Тесты выхода из аккаунта
│   └── test_ad_creation.py   # Тесты создания объявления
├── conftest.py               # Фикстура драйвера Chrome
├── helpers.py                # Генерация уникального email
├── test_data.py              # Тестовые данные (email, пароль, данные объявления)
├── README.md
└── .gitignore
```

## Покрытые сценарии

- **Регистрация**: успешная регистрация нового пользователя
- **Регистрация**: с невалидным email (не по маске)
- **Регистрация**: с данными уже существующего пользователя
- **Авторизация**: успешный вход в аккаунт
- **Выход**: успешный logout
- **Создание объявления**: неавторизованным пользователем (проверка модального окна)
- **Создание объявления**: авторизованным пользователем (проверка в профиле)

## Требования

- Python 3.10+
- Google Chrome (последняя версия)
- ChromeDriver (совместимый с установленным Chrome)

## Установка зависимостей

```bash
pip install selenium pytest
```

Или через `uv` (если используется):

```bash
uv sync
```

## Запуск тестов

```bash
# Все тесты
pytest tests/

# Конкретный файл
pytest tests/test_registration.py

# С подробным выводом
pytest tests/ -v
```

## Важно

Перед запуском тестов убедись, что в `test_data.py` указаны корректные данные существующего пользователя (`EXISTING_USER_EMAIL`, `EXISTING_USER_PASSWORD`). Эти данные используются в тестах на **Login**, **Logout** и **Создание объявления**.
